## 策略哲学

V1.0（少即是多）的三层过滤在趋势明确的行情中表现出色，但在震荡市中容易被反复打止损。

核心问题：**EMA 排列确认了趋势方向，但没有确认趋势是否正在持续。**

V2.0 增加了 ADX 过滤层，大幅改善了震荡市表现。BTC 365 天回测（+5.87%，胜率 31.9%，最大回撤 -7.11%）验证了方向正确，但暴露出两个问题：

1. **盈亏比偏低（0.46）**：TP2 = 5×ATR 不够远，盈利交易的平均收益被亏损抵消
2. **ADX = 25 阈值仍然偏松**：部分"趋势形成中"的假信号仍能通过

V2.1 针对以上问题进行了精确修复。

V2.2 基于 ETH 365 天回测（+19.06%，胜率 37%，盈亏比 0.99，最大回撤 -20.57%）发现：
- 趋势反转期 ADX 滞后导致连续止损（2025年11月连续 5 笔亏损）
- 需要更敏感的退出机制和连续止损保护

V2.2 新增三项保护措施：
1. **ADX 退出保护收紧**：ADX 从 ≥35 回落至 <25 时退出（更敏感）
2. **连续止损保护**：连续 3 笔亏损后暂停 24 小时
3. **EMA 距离监控**：EMA21 与 EMA55 距离 <0.5×ATR 时视为趋势衰竭信号

> 四层全过才出手。大多数时候不满足，**这正是它的优势。**

## 执行节奏

每 **2 小时** 触发一次检查循环。

---

## Step 1 · 第一层过滤：EMA 趋势方向

> *"方向错了，越努力越错。方向对了，等待就是收益。"*

调用 `market_get_candles` 获取标的的 **4h K线**，近 100 根。

计算三条 EMA：
- **EMA21**（短期动量）
- **EMA55**（中期趋势）
- **EMA200**（长期方向）

**第一层判断——只有两个结论：**

✅ **看多环境**：EMA21 > EMA55 > EMA200，且 EMA55 斜率向上（当前值 > 10根K线前的值）

✅ **看空环境**：EMA21 < EMA55 < EMA200，且 EMA55 斜率向下

❌ **混乱环境**：以上两种情况均不满足 → **本轮直接跳过，跳过全部后续分析**

**只有明确的环境才允许进入第二层过滤。模糊就是否定。**

## Step 2 · 第二层过滤：ADX 趋势持续性确认

> *"趋势的方向确认了，但趋势是否真的在持续？ADX 回答这个问题。"*

调用 `market_get_candles` 获取标的的 **4h K线**，计算 **ADX(14)**。

**计算方法（纯 Python / JavaScript）：**
```
TR  = max(H - L, |H - CL|, |L - CL|)   // CL = 前收盘价
+DM = H - H_prev  (仅当 > L - L_prev 且 > 0)
-DM = L_prev - L  (仅当 > H - H_prev 且 > 0)

+DI = 100 × EMA(+DM, 14) / EMA(TR, 14)
-DI = 100 × EMA(-DM, 14) / EMA(TR, 14)

DX  = 100 × |+DI - (-DI)| / (+DI + (-DI))
ADX = EMA(DX, 14)
```

**ADX 判断标准（V2.1 更新）：**

| ADX 值 | 市场状态 | 处理方式 |
|---|---|---|
| ADX < 20 | **无趋势**（震荡/横盘）| ❌ 跳过：市场方向不明 |
| 20 ≤ ADX < 30 | **趋势形成中** | ❌ 跳过：趋势不够强，信号过早 |
| **30 ≤ ADX < 45** | **趋势确认** | ✅ 允许进入第三层，仓位 10% |
| **ADX ≥ 45** | **极强趋势** | ✅ 允许进入第三层，仓位降至 5% |

> ⚠️ **V2.1 变更：阈值从 25 上调至 30。** V2.0 回测显示 ADX 25-30 区间的交易胜率低于整体，将阈值提升至 30 可过滤这部分低质量信号。

**为什么选择 30？**
- V2.0 回测（BTC 365天）中，ADX 25-30 区间入场交易胜率低于 30%
- ADX ≥ 30 时趋势已从"形成中"进入"确立"阶段，假突破概率显著降低
- 代价是减少约 15-20% 的交易机会，但保留的都是高质量信号

**动态仓位分档（V2.1 新增）：**

| ADX 范围 | 最大仓位上限 | 理由 |
|---|---|---|
| 30 ≤ ADX < 35 | 账户净值 × 5% | 趋势刚确认，保守入场 |
| 35 ≤ ADX < 45 | 账户净值 × 10% | 趋势确立，标准仓位 |
| ADX ≥ 45 | 账户净值 × 5% | 极强趋势但波动过大，缩仓保护 |

**额外保护：ADX 持续下降时强制退出（V2.2 更新）**
- 若持仓期间 ADX 从高位（≥ 35）回落至 < 25 → **立即平仓**（V2.1 是 < 20，V2.2 收紧至 < 25）
- 这比 EMA 反转止损更早发出警告信号
- **原因**：ETH 回测显示 ADX 从 35 跌至 20 过程中已损失 -15%，提前至 25 可减少回撤

## Step 3 · 第三层过滤：市场结构入场时机

> *"不要在别人跑得最快的时候跟上去。要在别人停下来喘气的时候，悄悄站到起跑线。"*

调用 **1h K线**，在第一层（EMA）和第二层（ADX）确认的方向基础上，寻找结构性入场机会。

**3.1 识别近期摆动高低点**

扫描近 50 根 1h K线，识别有效摆动点（左右各 3 根K线为极值）：
- 近期摆动低点序列：SL1（最近）、SL2（次近）、SL3（再次近）
- 近期摆动高点序列：SH1（最近）、SH2（次近）、SH3（再次近）

**3.2 结构条件判断**

**看多环境下，寻找看多结构入场信号（满足任意一项）：**

- **高低点上移（HL）**：SL1 > SL2（最新低点高于前低）且当前价格在 SL1 以上 → 回调未破结构，趋势延续
- **结构突破（BOS）**：当前1h K线收盘价突破 SH1（前高）→ 结构向上突破，新的入场机会

**看空环境下，寻找看空结构入场信号（满足任意一项）：**

- **低高点下移（LH）**：SH1 < SH2（最新高点低于前高）且当前价格在 SH1 以下 → 反弹未破结构，趋势延续
- **结构突破（BOS）**：当前1h K线收盘价跌破 SL1（前低）→ 结构向下突破，新的入场机会

❌ **以下情况放弃本轮**：
- 结构信号出现时，价格已运行超过 2.5 × ATR14（错过最佳入场窗口）
- 结构信号与第一层 EMA 方向相反

## Step 4 · 第四层过滤：ATR 动量确认

> *"引擎没有启动，踩油门也没用。动量不在，信号没有意义。"*

调用近期 **1h K线** 计算 ATR(14)。

**ATR 三档判断：**

| ATR 状态 | 判断标准 | 处理方式 |
|---|---|---|
| **过低（市场睡着了）** | 当前 ATR < 过去 30 根K线 ATR 均值的 **0.7 倍** | 跳过：市场没有方向性 |
| **正常（市场清醒）** | 当前 ATR 在均值的 **0.7 ~ 1.8 倍**之间 | ✅ 允许入场，标准仓位 |
| **过高（市场发狂）** | 当前 ATR > 过去 30 根K线 ATR 均值的 **1.8 倍** | 跳过：波动过大，止损难以设置 |

**只有"正常"状态才允许入场。市场太平静和太疯狂，都不是好时机。**

额外检查：
- 调用 `market_get_funding_rate`：资金费率绝对值 > 0.1%，跳过（市场方向已过度拥挤）

## Step 6 · 执行下单（仅当四层全部通过）

> *"决定已经在四层过滤里做完了。下单只是按钮。"*

调用 `swap_place_order`：

```
instId  = "<标的>-USDT-SWAP"   // 例：BTC-USDT-SWAP, ETH-USDT-SWAP
side    = <buy 或 sell>
ordType = "market"
sz      = <基于风控计算的张数>
tag     = "agentTradeKit"   ← 必填
```

**仓位计算（V2.1 动态仓位）：**

```
单笔风险金额 = 账户净值 × 1.5%
止损距离 = 2 × ATR14（入场时的ATR值）
计划张数 = 单笔风险金额 ÷ 止损距离

// 动态仓位上限（根据 ADX 分档）
if ADX < 35:   最大仓位 = 账户净值 × 5%
elif ADX < 45: 最大仓位 = 账户净值 × 10%
else:          最大仓位 = 账户净值 × 5%

计划张数 = min(计划张数, 最大仓位对应的张数)
```

## Step 7 · 止损与止盈（V2.1 更新）

> *"设好规则，然后让规则运行。不需要盯盘，不需要焦虑。"*

**止损**（调用 `swap_place_algo_order`，V2.2 更新）：

| 方向 | 止损价 | 逻辑 | V2.2 变更 |
|---|---|---|---|
| 做多 | 入场价 - 2 × ATR14 | 标准止损 | ADX ≥ 40 时扩大至 2.5×ATR |
| 做空 | 入场价 + 2 × ATR14 | 标准止损 | ADX ≥ 40 时扩大至 2.5×ATR |

> **V2.2 动态止损**：ADX ≥ 40（极强趋势）时，止损从 2×ATR 扩大至 **2.5×ATR**。
> 理由：极强趋势中价格波动更大，给趋势更多空间，避免被正常回调洗出。

**止盈（分两档，V2.1 更新 TP2）：**

| 档位 | 目标价 | 平仓比例 | V2.1 变更 |
|---|---|---|---|
| **TP1** | 入场价 ± 3 × ATR14 | 60% 仓位 | 不变 |
| **TP2** | 入场价 ± **6** × ATR14 | 剩余 40% | **5× → 6×** ⭐ |

TP1 触发后：止损移至**入场价（保本）**。

> ⚠️ **V2.1 变更：TP2 从 5×ATR 调整为 6×ATR。** V2.0 回测中盈亏比仅 0.46（平均盈利 ≈ 平均亏损），TP2 过早导致 40% 仓位的盈利空间不足。拉远 TP2 至 6×ATR，让趋势跑得更远，提升整体盈亏比。

**就这样。没有更多。**

## Step 8 · 持仓监控（V2.2 更新）

每次循环检查五件事：

1. **ADX 是否持续下降？** 若 ADX 从 ≥ 35 回落至 < 25 → **立即平仓全部仓位**（V2.2 收紧，原 < 20）
2. **EMA 方向是否反转？** 若 EMA21 穿越 EMA55 且方向反转 → 立即平仓全部仓位
3. **EMA 距离是否收窄？** 若 EMA21 与 EMA55 距离 < 0.5 × ATR14 → **趋势衰竭信号，减仓 50%**（V2.2 新增）
4. **连续止损保护**：若过去 3 笔交易全部亏损 → **暂停交易 24 小时**（V2.2 新增）
5. **止盈/止损单是否正常挂载？** 若未挂载 → 立即补挂

其余情况，**什么都不做，让策略运行。**

### V2.2 新增保护机制详解

**EMA 距离监控（趋势衰竭早期预警）**
```
EMA 距离 = |EMA21 - EMA55|
若 EMA 距离 < 0.5 × ATR14：
  → 短期与中期趋势趋于一致， momentum 减弱
  → 减仓 50%，保留部分仓位观察
  → 若距离继续收窄至 < 0.3 × ATR14，全部平仓
```

**连续止损保护（防止回撤扩大）**
```
维护最近 3 笔交易结果队列
若队列中亏损次数 = 3：
  → 触发连续止损保护
  → 暂停新开仓 24 小时
  → 已有持仓正常监控，不受限制
  → 24 小时后重置计数器
```

## 策略的自我叙述

只做确定趋势，是少即是多的进化版。

V2.0 学会了"确认趋势是否在持续"。
V2.1 学会了"只在趋势真正确立时下重注"。
V2.2 学会了"在趋势衰竭前及时撤退"。

它现在会：
- 在 ADX 从高位回落时提前离场（而不是等到 < 20）
- 在连续亏损后主动休息（而不是继续硬扛）
- 在 EMA 距离收窄时感知趋势衰竭（比 EMA 交叉更早）
- 在极强趋势中给止损更多空间（避免被正常回调洗出）

ADX < 30 时，它什么都不做——即使 EMA 排列完美，即使结构信号诱人。
ADX 30-35 时，它只下小注——趋势刚确认，还需要观察。
ADX 35-45 时，它下标准仓位——趋势已确立，是出击的好时机。
ADX ≥ 45 时，它又缩回去——趋势太强反而危险。

第一层说："方向是对的。"
第二层说："趋势真的在持续，而且够强。"
第三层说："入场时机也对了。"
第四层说："市场有足够的动力。"

四个问题全部得到肯定回答时，它不犹豫。
