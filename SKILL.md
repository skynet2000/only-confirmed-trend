# 策略名称
只做确定趋势（Only Confirmed Trend） V2.0

---

> *"趋势只有在被确认之后才值得追随。"*
> *升级自：少即是多（Less-is-More）V1.0*

---

## 策略哲学

V1.0（少即是多）的三层过滤在趋势明确的行情中表现出色，但在震荡市中容易被反复打止损。

核心问题：**EMA 排列确认了趋势方向，但没有确认趋势是否正在持续。**

本策略在 EMA 趋势方向（第一层）之后，新增 **ADX > 25 趋势持续性确认（第二层）**——只有当市场真正处于趋势状态时，才允许第三层（结构）和第四层（ATR）的过滤继续。

> 四层全过才出手。大多数时候不满足，**这正是它的优势。**

---

## 执行节奏

每 **2 小时** 触发一次检查循环。

---

## Step 1 · 第一层过滤：EMA 趋势方向

> *"方向错了，越努力越错。方向对了，等待就是收益。"*

调用 `market_get_candles` 获取标的的 **4h K线**，近 100 根。

计算三条 EMA：
- **EMA21**（短期动量）
- **EMA55**（中期趋势）
- **EMA200**（长期方向）

**第一层判断——只有两个结论：**

✅ **看多环境**：EMA21 > EMA55 > EMA200，且 EMA55 斜率向上（当前值 > 10根K线前的值）

✅ **看空环境**：EMA21 < EMA55 < EMA200，且 EMA55 斜率向下

❌ **混乱环境**：以上两种情况均不满足 → **本轮直接跳过，跳过全部后续分析**

**只有明确的环境才允许进入第二层过滤。模糊就是否定。**

---

## Step 2 · 第二层过滤：ADX 趋势持续性确认

> *"趋势的方向确认了，但趋势是否真的在持续？ADX 回答这个问题。"*

调用 `market_get_candles` 获取标的的 **4h K线**，计算 **ADX(14)**。

**计算方法（纯 Python / JavaScript）：**
```
TR  = max(H - L, |H - CL|, |L - CL|)   // CL = 前收盘价
+DM = H - H_prev  (仅当 > L - L_prev 且 > 0)
-DM = L_prev - L  (仅当 > H - H_prev 且 > 0)

+DI = 100 × EMA(+DM, 14) / EMA(TR, 14)
-DI = 100 × EMA(-DM, 14) / EMA(TR, 14)

DX  = 100 × |+DI - (-DI)| / (+DI + (-DI))
ADX = EMA(DX, 14)
```

**ADX 判断标准：**

| ADX 值 | 市场状态 | 处理方式 |
|---|---|---|
| ADX < 20 | **无趋势**（震荡/横盘）| ❌ 跳过：市场方向不明 |
| 20 ≤ ADX < 25 | **趋势形成中** | ❌ 跳过：趋势不够强，信号过早 |
| **ADX ≥ 25** | **趋势确认** | ✅ 允许进入第三层 |
| ADX ≥ 40 | **极强趋势** | ⚠️ 注意：波动可能过大，可适当缩小仓位 |

**为什么选择 25？**
- ADX < 25 时，价格通常在区间内震荡，进场容易被假突破打止损（正是 V1.0 在震荡市亏损的核心原因）
- ADX ≥ 25 是经典的趋势确认阈值（ Welles Wilder 原著推荐）
- ADX ≥ 40 表示趋势极强，此时可以交易，但需注意止盈距离可能过大

**额外保护：ADX 持续下降时强制退出**
- 若持仓期间 ADX 从高位（≥ 30）回落至 < 20 → **立即平仓**
- 这比 EMA 反转止损更早发出警告信号

---

## Step 3 · 第三层过滤：市场结构入场时机

> *"不要在别人跑得最快的时候跟上去。要在别人停下来喘气的时候，悄悄站到起跑线。"*

调用 **1h K线**，在第一层（EMA）和第二层（ADX）确认的方向基础上，寻找结构性入场机会。

**3.1 识别近期摆动高低点**

扫描近 50 根 1h K线，识别有效摆动点（左右各 3 根K线为极值）：
- 近期摆动低点序列：SL1（最近）、SL2（次近）、SL3（再次近）
- 近期摆动高点序列：SH1（最近）、SH2（次近）、SH3（再次近）

**3.2 结构条件判断**

**看多环境下，寻找看多结构入场信号（满足任意一项）：**

- **高低点上移（HL）**：SL1 > SL2（最新低点高于前低）且当前价格在 SL1 以上 → 回调未破结构，趋势延续
- **结构突破（BOS）**：当前1h K线收盘价突破 SH1（前高）→ 结构向上突破，新的入场机会

**看空环境下，寻找看空结构入场信号（满足任意一项）：**

- **低高点下移（LH）**：SH1 < SH2（最新高点低于前高）且当前价格在 SH1 以下 → 反弹未破结构，趋势延续
- **结构突破（BOS）**：当前1h K线收盘价跌破 SL1（前低）→ 结构向下突破，新的入场机会

❌ **以下情况放弃本轮**：
- 结构信号出现时，价格已运行超过 2.5 × ATR14（错过最佳入场窗口）
- 结构信号与第一层 EMA 方向相反

---

## Step 4 · 第四层过滤：ATR 动量确认

> *"引擎没有启动，踩油门也没用。动量不在，信号没有意义。"*

调用近期 **1h K线** 计算 ATR(14)。

**ATR 三档判断：**

| ATR 状态 | 判断标准 | 处理方式 |
|---|---|---|
| **过低（市场睡着了）** | 当前 ATR < 过去 30 根K线 ATR 均值的 **0.7 倍** | 跳过：市场没有方向性 |
| **正常（市场清醒）** | 当前 ATR 在均值的 **0.7 ~ 1.8 倍**之间 | ✅ 允许入场，标准仓位 |
| **过高（市场发狂）** | 当前 ATR > 过去 30 根K线 ATR 均值的 **1.8 倍** | 跳过：波动过大，止损难以设置 |

**只有"正常"状态才允许入场。市场太平静和太疯狂，都不是好时机。**

额外检查：
- 调用 `market_get_funding_rate`：资金费率绝对值 > 0.1%，跳过（市场方向已过度拥挤）

---

## Step 5 · AI 四层确认总结（决策核心）

> *"四个绿灯同时亮，才出发。一个红灯，等待。"*

```
【四层过滤检查表】

第一层 · EMA 趋势：
□ 趋势方向：[看多 / 看空]
□ EMA 排列：[21 > 55 > 200 / 21 < 55 < 200]
□ EMA55 斜率：[向上 / 向下]
→ 结论：[✅ 通过 / ❌ 不通过]

第二层 · ADX 趋势持续性：
□ ADX(14) 当前值：[X.XX]
□ ADX 判定： [< 20 无趋势 / 20-25 形成中 / >=25 确认 / >=40 极强]
→ 结论：[✅ 通过 / ❌ 不通过]

第三层 · 市场结构：
□ 结构信号类型：[HL回调 / BOS突破 / LH反弹 / 向下BOS]
□ 信号K线距当前：[X根K线前]
□ 是否在最佳入场窗口（< 2.5ATR）：[是 / 否]
→ 结论：[✅ 通过 / ❌ 不通过]

第四层 · ATR 动量：
□ 当前 ATR：[X.XX]
□ ATR 均值：[X.XX]
□ ATR 倍率：[X.XX]（目标范围：0.7 ~ 1.8）
□ 资金费率：[X.XXXX%]（上限：±0.1%）
→ 结论：[✅ 通过 / ❌ 不通过]

最终决策：

```

**如果任何一层为 ❌，最终决策自动为"等待"，不得下单。**

## Step 6 · 执行下单（仅当四层全部通过）

> *"决定已经在四层过滤里做完了。下单只是按钮。"*

调用 `swap_place_order`：

```
instId  = "<标的>-USDT-SWAP"   // 例：BTC-USDT-SWAP, ETH-USDT-SWAP
side    = <buy 或 sell>
ordType = "market"
sz      = <基于风控计算的张数>
tag     = "agentTradeKit"   ← 必填
```

**仓位计算：**

```
单笔风险金额 = 账户净值 × 1.5%
止损距离 = 2 × ATR14（入场时的ATR值）
计划张数 = 单笔风险金额 ÷ 止损距离
最大仓位上限 = 账户净值 × 10%（取两者较小值）
ADX 调整：当 ADX >= 40 时，最大仓位上限降至 账户净值 × 5%
```

---

## Step 7 · 止损与止盈

> *"设好规则，然后让规则运行。不需要盯盘，不需要焦虑。"*

**止损**（调用 `swap_place_algo_order`）：

| 方向 | 止损价 | 逻辑 |
|---|---|---|
| 做多 | 入场价 - 2 × ATR14 | 两个 ATR 内未被否定，趋势仍有效 |
| 做空 | 入场价 + 2 × ATR14 | 同上 |

**止盈（分两档）：**

| 档位 | 目标价 | 平仓比例 |
|---|---|---|
| **TP1** | 入场价 ± 3 × ATR14 | 60% 仓位 |
| **TP2** | 入场价 ± 5 × ATR14 | 剩余 40% |

TP1 触发后：止损移至**入场价（保本）**。

**就这样。没有更多。**

---

## Step 8 · 持仓监控

每次循环检查三件事：

1. **ADX 是否持续下降？** 若 ADX 从 ≥ 30 回落至 < 20 → **立即平仓全部仓位**（比 EMA 反转更早退出）
2. **EMA 方向是否反转？** 若 EMA21 穿越 EMA55 且方向反转 → 立即平仓全部仓位
3. **止盈/止损单是否正常挂载？** 若未挂载 → 立即补挂

其余情况，**什么都不做，让策略运行。**

## 风控总则

```
// [P1] 单笔最大风险 1.5%，不因任何理由提高
// [P2] 当日回撤超 4.5% 停止新开仓（三次止损即触发）
// [P3] 最多同时持有 1 个仓位（只做确定趋势的核心）
// [P4] 开仓后 60 秒内完成止损挂单
// [P5] 严禁在亏损时加仓，严禁移动止损扩大亏损
// [P6] 单日最多触发 3 次完整入场机会（多于此则说明市场混乱，降频）
// [P7] ADX >= 40 时，最大仓位上限降至 5%（极强趋势也需控制）
```

第一层说："方向是对的。"

四个问题全部得到肯定回答时，它不犹豫。
